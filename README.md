# Git 工作流安全检查

在提交或上传代码前做一次安全体检，减少误操作和敏感文件泄露。

## 它能解决什么问题

提交代码前，我们很容易忽略一些小问题：

- 当前是不是在错误的分支；
- 有没有忘记提交或不该提交的文件；
- 用户名和邮箱是否配置正确；
- `.env`、私钥或账号凭据是否混进了代码；
- 分支名称是否符合团队约定。

这个 Skill 会提前列出风险，但不会替你提交、推送或修改代码。

## 你可以这样使用

安装：

```bash
npx skills add https://github.com/dyc0616c-oss/git-safety-check
```

然后直接对 Agent 说：

```text
提交代码前，帮我检查这个项目有没有风险。
```

```text
检查当前分支、未提交文件和可能泄露的敏感文件。
```

```text
告诉我哪些问题必须先处理，但不要修改代码。
```

## 适合谁

- 经常使用 Git，但担心误提交文件的人；
- 希望统一提交规范的小团队；
- 准备把项目上传到 GitHub 的个人开发者。

## 它不会做什么

- 不会自动提交或推送代码；
- 不会切换、删除或合并分支；
- 不会修改 Git 配置；
- 只读取当前仓库的信息并给出检查结果。

## 来源

本项目参考 bdgit-governance 的治理思路重新实现，并移除了组织内部规则与服务。
